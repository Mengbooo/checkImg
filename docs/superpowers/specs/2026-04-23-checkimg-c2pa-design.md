# C2PA 图片检测工具 - 设计文档

**日期**: 2026-04-23
**项目**: checkImg - AI生成图片检测工具

## 1. 概述

一个纯前端的响应式网页工具，通过 C2PA (Coalition for Content Provenance and Authenticity) 元数据解析，判断上传的图片是否为 AI 生成。

## 2. 技术栈

| 类别 | 选择 |
|------|------|
| 构建工具 | Vite |
| 框架 | React 18 |
| C2PA 解析 | c2pa npm 包（浏览器端） |
| 样式方案 | Paper Design 设计系统 |
| 部署方式 | 静态单页应用 (SPA) |

## 3. 功能范围

### 核心功能（必须）
- **单图检测**: 用户上传一张图片，立即显示判断结果
- **简洁判断**: 仅显示 "AI生成" / "原创/未知" / "无C2PA元数据" 三种状态
- **图片格式支持**: JPEG、PNG、WebP

### 扩展功能（如可实现）
- **C2PA 详情展示**: 内联展开显示元数据详情（生成工具、参数、来源等）

## 4. 页面结构

```
App
├── Header
│   └── 标题 + 简短说明
├── ImageUploader
│   └── 拖拽/点击上传区域
├── ResultDisplay
│   ├── StatusBadge (AI生成 / 原创 / 无C2PA)
│   └── DetailPanel (C2PA 元数据详情，可展开)
```

## 5. 交互流程

1. 用户拖拽或选择图片文件
2. 前端读取文件并调用 c2pa 库解析
3. 根据 C2PA 数据判断并展示结果
4. 用户可展开查看详细元数据

## 6. 状态定义

| 状态 | 展示内容 |
|------|----------|
| 空闲 | 上传提示区 |
| 加载中 | loading 状态 |
| AI生成 | 红色警告标识 + "AI生成" |
| 原创/已知来源 | 绿色标识 + "原创/已知来源" |
| 无C2PA元数据 | 灰色标识 + "无元数据" |
| 解析错误 | 错误提示 |

## 7. 响应式设计

遵循 Paper Design 设计系统：
- 移动端优先
- 断点: 640px (sm), 768px (md), 1024px (lg)
- 核心布局: 单列居中，最大宽度 640px

## 8. 非功能需求

- **性能**: 图片解析在浏览器端完成，不上传到服务器
- **隐私**: 图片仅在本地处理，不发送至任何外部服务
- **兼容性**: 现代浏览器 (Chrome, Firefox, Safari, Edge 最新版本)

## 9. 后续扩展方向

- 批量图片检测
- 图片 URL 输入检测
- 更多图片格式支持 (AVIF, TIFF 等)
- 导出检测报告
